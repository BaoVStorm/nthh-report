# Report “Weekly Meetup with the FCAJ Community”

### Event Objectives

* Share learning methods, self-development approaches, and career orientation for students in the FCAJ community.
* Introduce how to use Prompt Engineering to improve the output quality of LLMs.
* Provide a new perspective on the BMAD Method, from free-form prompting to context engineering and Agile AI-oriented development workflows.

### Speaker List

* **Long Huynh** - Solution Architect, FCAJ
* **Nguyen Tuan Thinh** - DevOps/Cloud Engineer, FCAJ
* **Khang Nguyen** - Solution Architect, CloudKinetics
* **Thao Nguyen** - GenAI Engineer, VIB

### Key Highlights

#### Personal Learning Method

* The sharing focused on how to create learning motivation by increasing dopamine in a healthy way, for example rewarding oneself after completing a task.
* One practical method mentioned was the 2-minute rule: if something can be done within 2 minutes, it should be done immediately to avoid procrastination.
* This learning approach helps build the habit of taking quick action, reduce avoidance, and maintain more consistent learning energy.

#### Automated Prompt Engineering: Enhancing LLM Output Quality

* The session explained why Prompt Engineering is important: vague prompts often produce vague outputs, waste tokens, increase costs, and reduce productivity.
* A good prompt should include components such as role, instruction, context, input data, output format, examples, and constraints.
* The session also introduced advanced techniques such as Chain-of-Thought, Self-Consistency, Tree-of-Thoughts, RAG, and Role Prompting to improve the response quality of LLMs.

#### Career Orientation: Why Have You Not Started Working Yet?

* This part helped participants reflect on their career orientation through three circles: passion, benefits, and responsibility.
* Work does not only provide salary, but also brings experience, network, knowledge, and growth.
* Companies often evaluate candidates based on attitude, skills, experience, exposure, and personal qualities.

#### BMAD Method

* The BMAD Method changes the working approach from “one prompt does everything” to breaking tasks down in an Agile-oriented way with a clearer process.
* Instead of coding blindly without prior design, BMAD emphasizes having a PRD, system architecture, and role separation for each agent.
* This method helps separate context for each agent, reduce confusion in large projects, and support scaling based on domain complexity.

### Lessons Learned

* Effective learning does not only depend on willpower, but also requires suitable methods to reduce procrastination and maintain motivation.
* When working with LLMs, the clearer the prompt is with specific context and constraints, the easier it is to control the output.
* Career orientation should balance what I enjoy, what brings benefits, and the responsibilities I need to fulfill.
* The BMAD Method shows that AI is not only used for single Q&A tasks, but can also become part of an organized software development workflow.

### Application to Study and Work

* Apply the 2-minute rule and self-reward mechanism to reduce procrastination when studying AWS, working on projects, or writing reports.
* Use a prompt structure that includes role, task, context, input, output format, and constraints when working with ChatGPT or other LLMs.
* When building large projects, break work into clear tasks instead of asking AI to handle everything in a single prompt.
* Apply BMAD thinking to prepare PRDs, design systems, and separate context before starting to code.

### Event Experience

* The meetup provided a lot of practical knowledge, not only about technology but also about learning methods, career orientation, and effective use of AI.
* The Automated Prompt Engineering session helped me better understand how to communicate with AI to receive more accurate, structured, and time-saving results.
* The sharing about career development and the BMAD Method gave me more perspectives on self-development, project building, and better preparation for the real working environment.

#### Key takeaways

* To learn sustainably, it is necessary to build small, easy-to-follow habits and have a suitable motivation mechanism.
* Prompt Engineering is an important skill because input quality directly affects the output quality of AI.
* Work is not only about salary, but also an opportunity to gain experience, expand networks, increase knowledge, and develop oneself.
* When using AI for large projects, it is necessary to move from free-form prompting to context engineering and a structured workflow.
